请求 data：

### 1. Data

路径: Concatenate all values in the order in 路径. The restful 路径 in the form of `/test/{var1}/{var2}/` will be spliced according to the actual parameters filled in, for example: /sign/test/bb/aa

查询参数: Sorted by lexicographic order of key, concatenate all key=value. 示例: user名称=dfdfdf&password=ggg

请求体: Json: Operate as JSON 字符串 is not converted or sorted.

          x-www-form-urlencoded: Sorted by lexicographic order of key, concatenate all key=value. 示例: user名称=dfdfdf&password=ggg　

          form-data：Not supported.

If there are multiple data forms, re-concatenate in the order of 路径, 查询参数, and 请求体 to obtain the concatenate value of all data.

示例 of 路径:

          /future/api/v1/public/symbol/detail

          The above concatenated value is recorded as 路径

#### 示例 1. All parameters sent via 查询参数 字符串

            symbol=btc_usdt&side=BUY&类型=LIMIT&timeInForce=GTC&quantity=1&price=0.1

            The above concatenate value is recorded as 查询参数

#### 示例 2.All parameters send X-www-form-urlencoded request 请求体 字符串 via the request 请求体

请求 请求体 字符串 of json

            symbol=btc_usdt&side=BUY&类型=LIMIT&timeInForce=GTC&quantity=1&price=0.1

            The above concatenate value is recorded as 请求体

请求 请求体 字符串 of www-form-urlencoded

            `{"symbol" : "btc_usdt","side" : "BUY","类型":"LIMIT","timeInForce":"GTC","quantity":2,"price":39000}`

            The above concatenate value is recorded as 请求体

#### 示例 3.Mix to use 查询参数 字符串 and request 请求体 (form and json format)

查询参数String: symbol=btc_usdt&side=BUY&类型=LIMIT&timeInForce=GTC 上述拼接值记作 查询参数

requestBody: `{"quantity":2,"price":39000}` The above concatenate value is recorded as 查询参数

        The final concatenate value of the entire data is # concatenated with 路径, 查询参数, and 请求体 and form #路径, #查询参数, and #请求体. The finalconcatenate value is recorded asY=#路径#查询参数#请求体。

        注意:

            查询参数 without data, 请求体 with data:Y=#路径#请求体

            查询参数 with data, 请求体 without data:Y=#路径#查询参数

            查询参数 with data, 请求体 with data:Y=#路径#查询参数#请求体

### 2. 请求 header

X=”validate-appkey=3976eb88-76d0-4f6e-a6b2-a57980770085&validate-timestamp=1641446237201”

### 3. Obtain 签名

Finally, record the 字符串 that needs to be encrypted as sign=XY

Finally, encrypt the final concatenated value according to the following 方法 to obtain a signature.

signature=org.apache.commons.codec.digest.HmacUtils.hmacSha256Hex(secretkey, sign);

Put the generated signature in the request header, with validate-signature as the key and singature as the value.
