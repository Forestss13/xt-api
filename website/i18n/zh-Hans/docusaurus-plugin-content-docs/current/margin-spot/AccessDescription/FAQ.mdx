1. AUTH_105: The server verifies the request header parameters validate-timestamp (validTimeStamp) and validate-recvwindow (recvwindow).

The following rules must be followed: dealTimeStamp (server time when the request is processed, in milli秒) - validTimeStamp < recvwindow, otherwise AUTH_105 会返回.

To avoid this error, validate-timestamp recommends using the time when the request was sent, and it is measured in milli秒. The validate-recvwindow is set a little larger.
