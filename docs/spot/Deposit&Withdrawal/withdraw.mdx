**Type:** POST **Description:** `/v4/withdraw`

### Parameters

> ðŸ“Œ Note: The parameters are placed in the **request body** in JSON format.

| name | type | mandatory | default | description | ranges |
| --- | --- | --- | --- | --- | --- |
| currency | string | true | N/A | Currency name, from **Get supported currencies for deposit/withdrawal** API |  |
| chain | string | false | N/A | Transfer network name, from **Get supported currencies for deposit/withdrawal** API |  |
| clientOrderId | string | false | N/A | Custom client ID, regex: `^[a-zA-Z0-9_]{4,32}$` |  |
| amount | number | true | N/A | Withdrawal amount (including fee) |  |
| address | string | false | N/A | Withdrawal address |  |
| memo | string | false | N/A | Memo/Tag (required for some chains such as EOS) |  |
| toAccountId | number | false | N/A | Receiving user ID |  |

### Notes

- Parameters must be placed in the body as JSON.
- Limit flow rules: **1 request/second per apikey**.
- For EOS-like chains, `memo` must be provided.
- The returned `id` can be used to query withdrawal history later.

### Request Example

```json title="Request"
{
  "currency": "zb",
  "chain": "Ethereum",
  "amount": 1000,
  "address": "0xfa3abfa50eb2006f5be7831658b17aca240d8526",
  "memo": ""
}
```
